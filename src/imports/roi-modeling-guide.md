ROI MODELING LAYER

ROI must be derived from execution plan + validated metrics.

Never from storytelling.

1️⃣ ROI Input Requirements (Mandatory Before Calculation)

Each recommendation must provide:

Baseline metric

Target metric (30 / 60 / 90 days)

Timeline

Cost estimate

Confidence score

Feasibility score

If any missing → ROI locked.

2️⃣ ROI Core Formula

For Revenue Impact:

Projected Gain = (Target - Baseline) × Volume × Time Window

For Cost Reduction:

Savings = Time Saved × Cost per Hour × Period

For Risk Reduction:

Convert to probabilistic value:

Expected Risk Loss Avoided =
Risk Probability × Estimated Financial Exposure
3️⃣ Conservative Adjustment Layer (Safety Net)

This is critical.

Raw ROI = (Projected Gain - Investment Cost) / Investment Cost
Adjusted ROI = Raw ROI × (Confidence Score / 100)

If Confidence = 72%
Raw ROI = 300%
Displayed ROI = 216%

This protects credibility.

4️⃣ ROI Range Instead of Fixed Number

Always show:

Low Case → Based on 60% execution efficiency
Mid Case → Based on 80% execution efficiency
High Case → Based on 100% execution efficiency

Never show single ROI number.

5️⃣ Payback Period Formula
Payback Months =
Investment Cost / Monthly Net Gain

Displayed clearly.

6️⃣ Portfolio-Level ROI

After all recommendations:

Total Adjusted Portfolio ROI =
Sum(All Adjusted Gains) / Total Investment

But:
If dependencies exist → do not double count.

Dependency rule:

If A enables B,
only B gets full revenue credit.
A gets efficiency credit only.

Prevents stacking fantasy ROI.

7️⃣ ROI Display Structure

Each recommendation shows:

Investment

90-day gain projection

12-month projection

Payback timeline

Confidence-adjusted ROI %

Assumptions used

Portfolio shows:

Total capital required

Risk-adjusted return

Execution order impact curve
