Each domain receives:

problem_density_score (0–10)
impact_potential_score (0–10)
automation_feasibility_score (0–10)
risk_exposure_score (0–10)
2️⃣ Recommendation Creation Rule

Create recommendation ONLY IF:

(problem_density_score ≥ 6)
AND
(impact_potential_score ≥ 6)

Otherwise:
No recommendation generated for that domain.

This prevents fluff.

3️⃣ Priority Ranking Formula (Portfolio Level)

For each generated recommendation:

priority_score =
(impact_potential × 0.4)
+ (automation_feasibility × 0.3)
+ (problem_density × 0.2)
- (risk_exposure × 0.1)

Sort descending.

This determines execution order.

Not opinion.
Math.

4️⃣ Portfolio Size Guardrail

Maximum:
5–7 recommendations per business.

If more qualify:
Keep top 7 by priority_score.

This prevents overwhelm.

5️⃣ Execution Sequencing Logic

Sequence rules:

Fix constraints before optimization

Fix operational bottlenecks before growth acceleration

Stabilize data before advanced AI automation

Reduce risk before scaling spend

This creates logical sprint order.
